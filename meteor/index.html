Meteor Framework

What, Why, Wat

High-level Architecture

It's Real-time

  • Collection Synchronization
  • Reactive data sources
  • Reactive templates

The Good

  • Looks cool
  • Might be good for certain types of apps
  • Fibers on the server (no callbacks, yay!)

The Bad

  • Always connected
  • Collection sync
  • Reactivity
  • Minimongo
  • Client
  • Server
  • Performance
  • Ecosystem

Always Connected

  1. Meteor applications are not crawlable
  2. No server-side templates
  3. Headless webkit on the server to render pages (duh)

Collection synchronization

  1. Paging not possible without ugly hacks
  2. Limited API
  3. No rate limiting
  4. Wasted bandwidth (mobile anyone?)

Reactivity

  1. Feels bolted-on in both client and server
  2. Mongo is not reactive - polling all the way

Minimongo

  1. No indexes
  2. No aggregations
  3. No map-reduce
  4. Slooooow

Client

  1. Flat variables namespace (Meteor.session)
  2. Flat templates namespace (Meteor.templates)
  3. No built-in routing
  4. No two-way data-binding
  5. Handlebars are too logic-less

Client: Continued

  1. Meteor.autorun() handle leaking
  2. Meteor-router double view triggering
  3. Lots of special cases in templates

Server

  1. Flat methods namespace (Meteor.methods)
  2. Ain't fast (minimongo + reactivity)
  3. No command-like API (no scripts!)
  4. No server-side templates

Performance

  1. 200 req/s on aws large for REST API
  2. 3s Chrome DOM lockup with 8k items in collection on Core i7
  3. So much faster without minimongo

Ecosystem

  1. Own package manager
  2. Didn't play nice with npm
  3. Not too many different extensions

Real experience

Talkalytics

  1. Customers
  2. Orders
  3. Real-time analytics

Problems

  1. Horrible client-side and server-side performance
  2. Amount of data during initial sync
  3. Browser DOM lockups
  4. Mobile clients were dying

Changes

  1. Got rid of collection synchronization
  2. REST-like API to work with data
  3. Client was dumbed down, minimongo not used
  4. Used MongoDB directly
  5. Reactivity is reduced to bare minimum

Conclusions

  1. Meteor is not mature
  2. Meteor is slow
  3. Meteor is not for mobile
  4. Meteor is not search engine friendly
  5. Meteor does not work if you have data
